# Agent Instructions & Repository Guidelines

> **CRITICAL**: Every new agent session should start by reading `agent-reference.md`.
>
> Then, read this file for repository-specific guidance.

## Before Starting Any Work

1. **Read the current state documents**:
   - ✅ [`agent-reference.md`](agent-reference.md) - Living automation policy (read FIRST)
   - ✅ [`TODO.md`](TODO.md) - Current sprint tasks and roadmap
   - ✅ [`ACHIEVED.md`](ACHIEVED.md) - Completed milestones and recent work
   - ✅ [`CHANGELOG.md`](CHANGELOG.md) - Version history and changes
   - ✅ [`updates/todo.md`](updates/todo.md) - Versioned roadmap (if present)
   - ✅ [`updates/completed.md`](updates/completed.md) - Completed tasks archive (if present)

2. **Check for blocked tasks**:
   - Look for tasks marked `[>]` (blocked/waiting)
   - Check dependencies before starting new work

3. **Review standards**:
   - [`docs/standards/`](docs/standards/) - Engineering & design guidelines
   - [`docs/standards/branding.md`](docs/standards/branding.md) - App naming and branding rules

## When Adding New Features or Fixes

### 1. Create or Link to TODO Item
- Add task to `TODO.md` with unique ID (e.g., `TODO-001`)
- Include: description, current status, dependencies, estimated effort
- Reference this ID in all related commits and PRs

### 2. Update as You Work
- Change status from `[ ]` (not started) → `[~]` (in progress) → `[x]` (completed)
- Keep description current with blockers and next steps
- Update before committing

### 3. Document Everything
Required for Definition of Done:
- ✅ Code implementation
- ✅ Tests added/updated
- ✅ Comments and JSDoc
- ✅ CHANGELOG.md entry
- ✅ ACHIEVED.md updated
- ✅ Commit message: `feat|fix|docs: description [TODO-XXX]`

### Example Workflow
```
1. Read TODO.md → See TODO-005 not started
2. Update TODO.md: TODO-005 from [ ] to [~]
3. Implement feature in code
4. Add tests
5. Update CHANGELOG.md with new features
6. Update ACHIEVED.md with completion date
7. Commit: "feat: implement version tracking for WhatsNew screen [TODO-005]"
8. Update TODO.md: TODO-005 from [~] to [x]
```

## Critical Rules

### ⚠️ Branding
- **Official name**: "Abridgd" (with 'd')
- **Never**: Use "abridged" (with 'e') in user-facing contexts
- **Technical**: Keep internal names as lowercase "abridged"
- **Reference**: `docs/standards/branding.md`

### ⚠️ Bundle Identifiers
- **iOS**: `com.mccalmedia.abridged`
- **Android**: `com.mccal.abridged`
- **Do not change** without approval from tech lead
- **Reason**: Registered on Apple Developer & Google Play

### ⚠️ Version Management
- Update `app.json` → `version` field for all releases
- Follow Semantic Versioning: MAJOR.MINOR.PATCH
- Update `CHANGELOG.md` BEFORE merging
- `eas.json` uses `appVersionSource: "remote"` - version source is `app.json`
 - Treat `updates/todo.md` as the version roadmap; add tasks by target version

### ⚠️ iOS/Android Folders
- **DO NOT** commit `ios/` or `android/` folders to git
- These are generated by `expo prebuild`
- If they exist locally, run: `npx expo prebuild --clean`
- See `.gitignore` for exclusions

### ⚠️ Branch Protection & CI
- Default branch is `master`; do not push directly.
- Open PRs from feature branches and wait for required checks:
   - `Lint docs` workflow must pass
   - `ci.yml` build/tests must pass
- Require at least one PR review; prefer Code Owners where applicable.

### ⚠️ Environment Variables
- API keys in `.env` file (excluded from git)
- Required: `EXPO_PUBLIC_PERPLEXITY_API_KEY`
- Add new keys to `.env.example` for documentation

## Common Tasks Checklist

### Adding a New Screen
- [ ] Create file: `src/screens/NewScreen.tsx`
- [ ] Add to `src/navigation/types.ts` (RootStackParamList or TabParamList)
- [ ] Import in `src/navigation/RootNavigator.tsx`
- [ ] Add `<Stack.Screen>` or `<Tab.Screen>` entry
- [ ] Add navigation option to Settings or other entry point
- [ ] Create tests: `src/screens/__tests__/NewScreen.test.tsx`
- [ ] Update TODO.md and ACHIEVED.md
- [ ] Update CHANGELOG.md

### Updating Theme/Colors
- Edit: `src/theme/colors.ts`, `src/theme/typography.ts`, or `src/theme/spacing.ts`
- Apply consistently across all screens
- Reference: ADR-0004 (Theming and semantic colors)

### Building for TestFlight
1. Ensure code is committed and clean
2. Run: `npx eas build --platform ios --profile production`
3. Once built, run: `npx eas submit --platform ios --latest`
4. See: `docs/deployment/testflight/ios-ipa-testflight.md`

### Running Tests
```bash
# All tests
npx jest --runInBand

# Specific test file
npx jest src/screens/__tests__/ArticleScreen.test.tsx --runInBand --verbose

# With coverage
npx jest --coverage
```

## Git Commit Message Format

```
type: description [TODO-XXX]

Body (optional): More details about the change.

Example:
feat: implement version tracking for What's New screen [TODO-005]

Stores lastSeenVersion in AsyncStorage to auto-show What's New
screen on first launch after app update. Compares with app.json version.
```

**Types**: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `chore`
