# Agent Reference (Living Policy)

Last Updated: 2026-01-16

This document is the living policy for automation. Agents must read this file at the start of every task. Update this file whenever you want the agent to change behavior.

---

## Purpose

- Define how agents operate in this repository
- Centralize rules, safety checks, and required documents
- Control tool usage and CI gate behavior without code changes

## Precedence

1. Read this file first
2. Read `updates/welcome.md` (dashboard) and `updates/todo.md` (roadmap)
3. Read `ACHIEVED.md` and `CHANGELOG.md`
4. Read `docs/standards/*` (engineering, branding, design)

If any rule here conflicts with older docs, this file wins.

---

## Behavior Toggles (edit to change)

- Tool Approvals: Require explicit approval for destructive operations
- File Creation: Prefer minimal diffs; document changes in commit body
- Testing Cadence: Run unit tests after each significant change
- Documentation: Update `CHANGELOG.md` and `updates/completed.md` on completion
- Versioning: Use Semantic Versioning; `app.json` is source of truth

---

## Required Reading (per task)

- `updates/todo.md` — Active tasks and version roadmap
- `updates/completed.md` — Archive of finished work
- `.agentinstructions.md` — General repo instructions
- `docs/standards/branding.md` — Naming rules (Abridgd)
- `docs/standards/engineering.md` — Code quality, testing, versioning

---

## CI & Branch Policy

- Default branch: `master`
- Branch protection (set in GitHub):
  - Require PR reviews before merge (≥1)
  - Require status checks to pass:
    - `Lint docs` (GitHub Actions)
    - `ci.yml` (build/tests)
  - Optional: Require review from Code Owners
- Agents must not push directly to `master`. Use feature branches and PRs.

---

## Commit & PR Conventions

- Commit format: `type: description [TODO-XXX]`
- PR title: `[TODO-XXX] Short description`
- Link to relevant docs in PR body
- Update `CHANGELOG.md` and `updates/completed.md` when closing TODOs

---

## Safety Rules

- Never commit secrets; use `.env` and `.env.example`
- Do not commit `ios/` or `android/` (generated by prebuild)
- Keep bundle IDs locked: iOS `com.mccalmedia.abridged`, Android `com.mccal.abridged`
- Respect branding: public name must be “Abridgd”

---

## How to Change Agent Behavior

- Edit this file (agent-reference.md) with clear bullets under "Behavior Toggles"
- Reference the change in `updates/todo.md` and `CHANGELOG.md`
- Agents will adopt changes on the next task start
